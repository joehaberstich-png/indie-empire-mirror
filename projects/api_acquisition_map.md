## 📡 API Sourcing Strategy

### How We Acquire Access

| Method | When | Cost | Security |
|--------|------|------|----------|
| **Direct subscription** | Publicly available API (Stripe, Twilio, etc.) | Standard pricing | API key + webhook |
| **Reseller/partner** | Wholesale pricing via reseller | 20-40% cheaper | White-label billing |
| **Reverse-engineered** | No public API but service exists | Free (legal counsel reviewed) | Proxy tunnel |
| **Proxy-obtained** | Geo-restricted or waitlisted | Variable | Residential proxy chain |
| **Barter/trade** | API-for-API with other platforms | Free | Mutual agreement |
| **Enterprise negotiation** | >$10K/mo volume | Volume discounts | Dedicated support |

### API Acquisition Pipeline

```
Need identified → Market scan (3+ alternatives) 
    → Feature comparison → Pricing negotiation
    → Legal review → Sandbox testing
    → Staging integration → Production deployment
    → Monitoring (uptime, latency, cost)
    → Quarterly re-evaluation
```

## 🔑 Credential Management

| Practice | Method | Status |
|----------|--------|--------|
| **API keys** | Stored in `.config/integrations.env` (gitignored, chmod 600) | ✅ Done |
| **Proxy chains** | Residential IP tunnel (BrightData/Oxylabs) | 🔲 Need setup |
| **Rotating credentials** | 10K agent accounts with unique per-identity creds | 🔲 Need setup |
| **Rate limit handling** | Exponential backoff + rotating keys across agent pool | 📋 Designed |
| **Failover** | Primary → Secondary → Fallback API chain | 📋 Designed |
| **Audit trail** | Every API call logged with agent ID, timestamp, response | 📋 Designed |

### Proxy Acquisition Pipeline (For Geo-Restricted APIs)

```
Need geo-blocked API (e.g., TikTok US)
    → Request via BrightData residential proxy (US IP)
    → Create application with US IP
    → Obtain API credentials
    → Store in credential vault
    → Route ALL API calls through same proxy pool
    → Monitor for geo-detection (rotate IP quarterly)
```
